# time_manager 开发框架文档（初步设计）

## 零、AI须知
- 需要确认的事项要与我确认清楚再执行
- 创建必要的文档和说明，将想法规范清楚
- 由于你只能写代码，需要告诉我需要执行什么相关操作，如部分测试和某些东西的安装

## 一、项目定位
- **开源的个人时间管理助手**，将未来一切事务汇集一处，用 AI 辅助规划与复盘。
- 核心实体：**日程、待办、长期目标**，专注时间管理，不加多余功能。
- 用户完全掌控数据与 AI 密钥，外部系统密码不上传服务器。
- AGPLv3 协议，人人可免费使用、修改、自部署。

## 二、功能模块清单
1. **日程管理**：含起止时间，或全天，可设置周期。课程、考试是特殊的日程。日程可设置推迟，会被记录并在报告中反馈。
2. **待办管理**：DDL、优先级、多种剩余时间视图。
3. **长期目标**：可记录进度，持续激励。
4. **准备时长/时段**：日程和待办均可选填准备分钟数，或指定具体准备时间段（指定的准备时段也会成为一个日程）。
5. **日历标识**：节日、生日等特殊日期。
6. **AI 自然语言提取**：提交文本，返回结构化任务，用户确认添加。
7. **AI 时间分配建议**：根据未来事项和空闲时间，生成准备时段建议。
8. **AI 早安推送**：每日定时生成人性化提醒与鼓励，推送至客户端，可设置系统推送、飞书推送等。
9. **外部数据抓取**：
   - 复旦 elearning / 教务系统：**纯客户端抓取**，密码不离开设备，结果上传同步。
   - 飞书群消息：通过飞书应用机器人接收，AI 提取任务。
   - 微信通知监听：**仅 Android 实验功能**，有封号风险，默认关闭。
10. **多端同步**：手机、桌面、Web 数据实时一致（WebSocket 推送）。
11. **本地提醒**：手机端可设闹钟/通知，桌面/Web 仅通知。
12. **月/年报告**：统计完成率、时间分布，可含 AI 总结并建议。
13. **客户端自动更新**：检测 GitHub Release 并引导升级。

## 三、技术架构
- **前端**：Flutter（Riverpod 状态管理），支持 Android/iOS/Windows/macOS/Web。
- **后端**：Python FastAPI + SQLAlchemy 异步 + Alembic 迁移。
- **数据库**：PostgreSQL 存储结构化数据。
- **缓存/队列**：Redis，用于缓存热点数据、ARQ 任务队列。
- **AI 网关**：LiteLLM，用户可自配任何兼容模型的 api_key，服务端加密存储。
- **实时推送**：WebSocket（FastAPI 原生支持）。
- **后台任务**：ARQ worker + ARQ cron（定时早安消息、报告生成等）。
- **部署**：Docker Compose 本地开发，未来单服务器生产部署，Nginx 反代。
- **CI/CD**：GitHub Actions 运行测试、构建多端产物、创建 Release。

## 四、数据库概要（仅表级关系）
- **users** - 用户基础信息。
- **events** - 日程，关联用户，支持 rrule、分类、外部来源标识。
- **event_preparation_periods** - 日程的明确准备时间段。
- **tasks** - 待办，含 deadline、优先级、状态、视图偏好。
- **task_preparation_periods** - 待办的明确准备时间段。
- **goals** - 长期目标及进度。
- **ai_configs** - 用户 AI 配置（provider、端点、加密 key、模型名）。
- **ai_templates** - 系统预设提示词模板。
- **notifications** - 已发送的早安消息等记录。
- **special_dates** - 节日、生日等日期标记。

*索引、约束、具体字段类型在开发阶段细化。*

## 五、API 模块划分（RESTful，/api/v1）
| 模块 | 主要端点 | 说明 |
|------|----------|------|
| 认证 | `/auth/register`, `/auth/login` | JWT 或 API Key |
| 日程 | `/events` CRUD + 批量上传 | 包含准备时段管理 |
| 待办 | `/tasks` CRUD + 批量上传 | 支持状态流转、视图偏好 |
| 目标 | `/goals` CRUD | 更新进度 |
| AI 配置 | `/ai/config` CRUD | 加密存储 api_key |
| AI 调用 | `/ai/extract`, `/ai/suggest-time`, `/ai/morning-message` | 文本提取、时间建议、生成早安 |
| 同步 | `/sync/upload` | 客户端抓取后的批量上传，去重逻辑 |
| 报告 | `/reports/monthly`, `/reports/yearly` | 聚合统计与 AI 总结 |
| 版本 | `/version` | 获取最新客户端版本信息 |
| 实时 | WebSocket `/ws/updates` | 推送数据变更事件 |

## 七、外部集成方案摘要
- **复旦系统**：客户端独立 Dart 包 `fudan_scraper` 负责登录抓取，密码仅内存使用，结果经 `/sync/upload` 同步。
- **飞书**：服务端接收应用机器人事件，AI 提取后直接创建任务。
- **微信**：Android 端 `NotificationListenerService` 实验模块，编译开关控制，UI 强提示风险。

## 八、定时任务与后台作业
- **早安推送**：ARQ cron，用户时区 7:00 触发，调 AI 生成 -> 存通知 -> WebSocket 推送。
- **报告预计算**：每月/年初凌晨生成并缓存。
- **清理**：定期清理旧通知与日志。

## 九、安全要点
- 密码：bcrypt 哈希。
- AI Key：服务端 Fernet AES 加密，密钥存环境变量。
- 外部系统密码：永不上传，仅客户端使用。
- API 通信强制 HTTPS，WebSocket 需 token 验证。

## 十、部署与 CI/CD
- **本地开发**：docker compose up 启动 API、Postgres、Redis、Worker、Cron。
- **生产**：单机 Docker Compose + Nginx SSL 反代。
- **CI**：PR 触发 lint、测试；**CD**：打 tag 时构建 Docker 镜像推送 ghcr，构建全平台 Flutter 产物，创建 GitHub Release 并上传。

## 十一、开发阶段（按模块推进，不设固定工期）
1. **骨架搭建**：FastAPI 项目、数据库模型、Docker 环境、Flutter 基本路由与认证。
2. **核心实体**：日程/待办/目标 CRUD + 多视图 + 准备时段 + 日历标识。
3. **AI 集成**：LiteLLM 封装、文本提取、时间建议、早安消息全链路。
4. **外部数据**：复旦抓取模块、飞书接入、微信实验模块、同步上传。
5. **实时与提醒**：WebSocket 推送、本地通知闹钟、自动更新。
6. **界面设计**：设计视图与界面结构，优化UI提升美感。（前期功能都采用简单入口，后期再考虑界面）
7. **报告与打磨**：月/年报、性能优化、开源文档、发布 1.0。
